# Plan: Author Book Content

**Feature**: 002-author-book-content
**Goal**: Implement the "Physical AI & Humanoid Robotics" book content.

## 1. File Structure Strategy

We will organize the documentation in `docusaurus-root/docs/` as follows:

```
docs/
├── intro.md                  # Introduction, Why Physical AI, Learning Outcomes
├── schedule.md               # Weekly Breakdown
├── hardware.md               # Hardware Requirements & Lab Options
├── assessments.md            # Assessments & Capstone
└── modules/                  # Folder for core modules
    ├── 01-nervous-system.md  # Module 1: ROS 2
    ├── 02-digital-twin.md    # Module 2: Gazebo & Unity
    ├── 03-ai-robot-brain.md  # Module 3: NVIDIA Isaac
    └── 04-vla.md             # Module 4: Vision-Language-Action
```

## 2. Sidebar Configuration (`sidebars.ts`)

We will configure the sidebar to be autogenerated or manually specified to ensure the correct order:
1. Introduction
2. Schedule
3. Modules (Dropdown)
4. Hardware Requirements
5. Assessments

### 3.5. Assessments (`assessments.md`)
